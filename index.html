AI Cloud Pricing Models Explained

Five distinct pricing models govern AI compute costs. Understanding each one determines whether you overpay by 2–5× or optimize correctly for your workload.

On-Demand
Base price
Pay per hour or second with zero commitment. Highest flexibility — start and stop anytime. Highest unit cost. Best for experiments, short training runs, and bursty inference.
Best for: Experiments
Reserved Instances
30–72% off
1-year or 3-year commitments to a specific GPU/instance type. Steep discounts for predictable workloads. Upfront or monthly payment options. Risk: wasted spend if usage drops.
Best for: Production workloads
Spot / Preemptible
60–91% off
Use unused cloud capacity at massive discounts. Can be interrupted with 2-minute warning (AWS) or 30-second notice (GCP). Requires fault-tolerant workloads with checkpointing.
Best for: Interruptible training
Savings Plans
20–40% off
Flexible commitment: pledge a $/hour spend for 1-3 years without locking into specific instance types. More flexible than reserved instances. AWS, GCP, and Azure all offer variants.
Best for: Mixed GPU workloads
Token-Based (LLM APIs)
Per-token
Pay per million tokens processed — input and output tokens billed separately. No GPU management. Covers OpenAI, Anthropic, Google Gemini, Mistral, and open-source inference APIs.
Best for: Inference-only workloads

AI Cloud Pricing Models – Complete Guide

How to choose the right pricing model for training, fine-tuning, and inference workloads at different scales.

On-demand vs reserved — when to switch

The break-even point for reserved instances is when your GPU utilization exceeds 40–50% of available hours. If you run an H100 for more than 10 hours/day consistently for 3+ months, a 1-year reserved instance saves money. Below that threshold, on-demand is cheaper because you avoid paying for idle hours you committed to.

Spot instances for AI training

Spot instances are viable for AI training only when your training code implements checkpointing — saving model state periodically so training can resume from the last checkpoint after interruption. PyTorch and JAX both support this natively. Most distributed training frameworks (DeepSpeed, FSDP) include built-in checkpoint support. Without checkpointing, spot interruptions waste all prior compute.

Hyperscalers vs specialist GPU clouds

AWS, GCP, and Azure offer deep integration with their broader ecosystems (storage, networking, MLOps tooling) but charge significant premiums for GPU compute. Specialist clouds — Lambda Labs, CoreWeave, RunPod, Vast.ai — offer GPU-only services at 40–70% lower on-demand prices with fewer managed services. Hyperscalers win on compliance, networking, and managed ML pipelines. Specialists win on raw GPU cost.

LLM API vs self-hosted GPU

LLM APIs (OpenAI, Anthropic, Gemini) abstract GPU management entirely — you pay per token and scale instantly with no infrastructure overhead. Self-hosted inference on your own GPU (or rented GPU) is cheaper above ~10M tokens/day per model. Below that threshold, API pricing is almost always more economical when you factor in engineering time, model deployment, and scaling complexity.

Pricing model decision tree

Start with on-demand to validate your workload. If it runs continuously for 3+ months, evaluate reserved. If it's interruptible training, evaluate spot. If your workload is pure inference without GPU management requirements, evaluate LLM APIs first — they're often cheaper than you expect at moderate scale and eliminate all infrastructure overhead.

Multi-cloud and arbitrage strategies

GPU prices vary significantly between providers and regions. An H100 on CoreWeave costs ~$2.49/hr; on AWS it costs ~$5.12/hr for the same hardware. Production systems increasingly use multi-cloud GPU routing — training on specialist clouds (lower cost) and serving inference on hyperscalers (better latency and compliance). Tools like SkyPilot and Flyte support multi-cloud GPU workload orchestration.

AI Cloud Pricing – FAQ

What are the main AI cloud pricing models?+
The five main AI cloud pricing models are: (1) On-demand — pay per hour, no commitment, highest cost; (2) Reserved instances — 1-3 year commitment for 30-72% discount; (3) Spot/preemptible — use unused capacity at 60-91% discount with interruption risk; (4) Savings plans — flexible $/hour commitment for 20-40% discount without locking into specific instance types; (5) Token-based — per-million-token billing for LLM APIs with no GPU management required.
Which cloud provider has the cheapest H100 GPU?+
As of May 2026, specialist GPU clouds offer significantly lower H100 prices than hyperscalers. Lambda Labs: ~$2.49/hr on-demand. CoreWeave: ~$2.65/hr. RunPod: ~$2.79/hr. AWS p4d.24xlarge (A100): ~$3.09/hr. AWS p5 (H100): ~$5.12/hr. GCP A3 (H100): ~$5.12/hr. The hyperscaler premium is real — you pay 2-3× more for H100s on AWS/GCP compared to Lambda or CoreWeave, in exchange for better ecosystem integration and compliance certifications.
When should I use spot instances for AI training?+
Use spot instances for AI training when: your training code implements checkpointing (saves state every N steps), your training job is long enough that the discount (60-91%) outweighs restart overhead, and you're not on a tight deadline where interruptions are unacceptable. Spot is ideal for pre-training, fine-tuning, and hyperparameter sweeps. Avoid spot for inference serving, time-sensitive training, and any job where losing 1+ hours of compute would be catastrophic without checkpointing.
How do I calculate LLM API costs?+
LLM API cost = (input tokens × input price per million) + (output tokens × output price per million). For example, GPT-4o at $2.50/M input tokens and $10.00/M output tokens: 500 input tokens + 200 output tokens per request × 1,000 requests/day = 500K input tokens + 200K output tokens daily = $1.25 + $2.00 = $3.25/day = ~$97.50/month. Use the LLM API calculator above to compare across providers automatically.
What is the difference between hyperscaler and specialist GPU clouds?+
Hyperscalers (AWS, GCP, Azure) offer GPU compute as part of broader cloud platforms with managed ML services (SageMaker, Vertex AI, Azure ML), VPC networking, compliance certifications (SOC2, HIPAA, FedRAMP), and global regions. They charge a significant premium for GPU compute. Specialist GPU clouds (Lambda Labs, CoreWeave, RunPod, Vast.ai) focus exclusively on GPU compute at 40-70% lower prices but offer fewer managed services and fewer compliance certifications. Choose specialists for cost-optimized training; consider hyperscalers when compliance, ecosystem integration, or managed MLOps matter.
Is it cheaper to use LLM APIs or host my own model?+
LLM APIs are cheaper than self-hosting below roughly 5-15M tokens/day depending on model size and GPU type. Above that volume, renting GPU capacity to run your own open-source model (Llama, Mistral, Qwen) becomes more economical. The break-even calculation: (GPU cost/hr × hrs/day × 30) vs (tokens/day × 30 × API price per token). Factor in engineering overhead — self-hosting requires infrastructure management, model updates, and scaling logic that add hidden costs.
